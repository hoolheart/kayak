## 1. 系统概述

### 1.1 什么是 Kayak

Kayak 是一个面向科学研究的设备管理与数据采集支撑平台。它帮助研究人员统一管理实验室中的各类测量设备，配置数据测点，并实时采集和监控设备数据。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **工作台 (Workbench)** | 一个独立的研究项目空间，包含一组设备及其测点。例如"高温材料实验"、"风洞测试"。 |
| **设备 (Device)** | 连接到平台的物理或虚拟测量仪器。支持 Virtual、Modbus TCP、Modbus RTU 三种协议。 |
| **测点 (Point)** | 设备上的具体数据采集位置，例如温度传感器的"通道1温度"、压力变送器的"出口压力"。 |
| **协议 (Protocol)** | 设备与 Kayak 平台之间的通信规则，决定数据如何传输。 |

### 1.3 平台架构

```
┌──────────────────────────────────────────────────┐
│                    Kayak Web UI                   │
│              (Flutter Web 前端界面)                │
├──────────────────────────────────────────────────┤
│                  Kayak Backend                    │
│          (REST API 服务 + 设备驱动管理)             │
├──────────────────────────────────────────────────┤
│     Virtual Driver  │  Modbus TCP  │  Modbus RTU  │
│    (虚拟设备驱动)     │  (网络驱动)  │  (串口驱动)   │
└──────────────────────────────────────────────────┘
```

### 1.4 支持的协议

| 协议 | 图标 | 说明 | 使用场景 |
|------|------|------|---------|
| **Virtual** | `developer_board` | 虚拟设备，用于模拟测试 | 开发测试、用户培训、无真实硬件时的功能验证 |
| **Modbus TCP** | `lan` | TCP/IP 网络通信协议 | 通过以太网连接的 Modbus 设备 |
| **Modbus RTU** | `usb` | 串口通信协议 (RS485/RS232) | 通过串口连接的 Modbus 设备 |

### 1.5 主题

Kayak 支持**浅色 (Light)** 和**深色 (Dark)** 两种主题，系统会根据您的操作系统主题设置自动匹配，也可在设置中手动切换。

## 2. 快速开始

### 2.1 系统要求

- **浏览器**: Chrome 90+、Edge 90+、Firefox 88+、Safari 15+
- **屏幕分辨率**: 建议 ≥ 1280 × 720
- **网络**: 能够访问 Kayak 后端服务 (默认 `http://localhost:8080`)

| 项目 | 值 |

## 3. 工作台管理

工作台是 Kayak 的核心组织单元。每个工作台对应一个独立的研究项目，包含相关的设备和测点。

### 3.1 工作台列表页

登录后，点击顶部导航栏的 **工作台** 或 Dashboard 的快捷操作卡片即可进入工作台列表页。

#### 视图切换

工作台列表支持两种视图模式：

- **网格视图 (⊞)**：以卡片形式展示工作台，适合少量工作台时快速浏览。
- **列表视图 (≡)**：以表格形式展示，包含名称、描述、设备数、状态、创建时间等列，适合工作台较多时的详细查看。

点击 App Bar 右侧的视图切换按钮可在两种视图间切换。

#### 搜索与筛选

- **搜索**：在搜索框中输入关键词，系统将按名称和描述进行模糊匹配。
- **筛选**：按工作台状态筛选（全部/活跃/已归档）。
- **排序**：按名称、创建时间、设备数量等字段排序。

### 3.2 创建工作台

#### 操作步骤

1. 点击页面右上角或底部的 **+ 创建工作台** 按钮。
2. 弹出创建对话框，包含以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 * | 是 | 工作台名称，例如"高温材料实验" |
| 描述 | 否 | 工作台的详细描述，支持多行文本 |

3. 填写完成后点击 **创建**，新工作台将立即出现在列表中。

```
│                                  │
│           [取消]   [创建]        │
└──────────────────────────────────┘
```

### 3.3 编辑工作台

1. 在工作台卡片上点击 ✏️ **编辑** 图标，或在列表视图中点击操作列的编辑按钮。
2. 弹出编辑对话框，名称和描述字段已预填当前值。
3. 修改后点击 **保存**。

### 3.4 删除工作台

> ⚠️ **警告**: 删除工作台将同时删除该工作台下所有设备和数据，此操作**不可撤销**。

1. 在工作台卡片上点击 🗑️ **删除** 图标。
2. 弹出确认对话框：

   ```
   │                                      │

   │                                      │
   │            [取消]   [确认删除]         │
   └──────────────────────────────────────┘
   ```
3. 确认后工作台和所有关联数据将被删除。

## 4. 设备管理

设备管理是 Kayak 平台的核心功能，支持三种工业协议的设备创建、编辑和连接管理。

### 4.1 创建设备

1. 在工作台详情页，点击左侧设备树上方的 **+ 添加设备** 按钮。
2. 弹出设备配置对话框，包含三个配置区域：

```
┌──────────────────────────────────────────────┐
│  创建设备                                 ✕  │
├──────────────────────────────────────────────┤
│                                              │
│  基本信息                                    │
│  ┌─ 设备名称 * ──────────────────────────┐  │
│  │ 输入设备名称...                        │  │
│  └───────────────────────────────────────┘  │
│  ┌─ 描述 ────────────────────────────────┐  │
│  │ 输入描述信息...                        │  │
│  └───────────────────────────────────────┘  │
│                                              │
│  协议配置                                    │
│  ┌─ 协议类型 * ── [Virtual ▼] ──────────┐  │
│  │                                       │  │
│  │  [协议参数表单 — 根据选择动态切换]       │  │
│  │                                       │  │
│  └───────────────────────────────────────┘  │
│                                              │
│  测点配置 (可选)                              │
│  ┌─ 测点表格 ────────────────────────────┐  │
│  │ 名称 │ 类型 │ 值 │ ...                 │  │
│  └───────────────────────────────────────┘  │
│                                              │
│        [⚡ 测试连接]      [取消] [创建]      │
└──────────────────────────────────────────────┘
```

#### 协议选择器

点击"协议类型"下拉框，可选择三种协议：

| 协议 | 图标 | 说明 |
|------|------|------|
| **Virtual** | `developer_board` | 虚拟设备，用于测试和模拟 |
| **Modbus TCP** | `lan` | TCP/IP 网络通信协议 |
| **Modbus RTU** | `usb` | 串口通信协议 (RS485/RS232) |

> ⚠️ **注意**: 编辑已有设备时，协议选择器将禁用，防止意外更改协议类型。如需切换协议，请删除设备后重新创建。

### 4.2 Virtual 协议配置

Virtual 协议用于创建虚拟设备，无需真实硬件即可进行数据模拟。适合开发测试和功能验证。

#### 配置参数

| 参数 | 说明 | 可选值 | 默认值 |
|------|------|--------|--------|
| **数据模式** | 虚拟设备的数据生成方式 | Random / Fixed / Sine / Ramp | Random |
| **数据类型** | 生成数据的类型 | Number / Integer / String / Boolean | Number |
| **访问类型** | 数据读写权限 | RO(只读) / WO(只写) / RW(读写) | RW |
| **最小值** | 数值范围下限 | 任意数值 | 0.0 |
| **最大值** | 数值范围上限 | 任意数值 | 100.0 |
| **固定值** | 固定模式的输出值 (仅 Fixed 模式) | 在最小值和最大值之间 | 50.0 |

#### 数据模式详解

| 模式 | 图标 | 行为 |
|------|------|------|
| **Random** | `shuffle` | 在最小值和最大值之间随机生成数值 |
| **Fixed** | `lock` | 始终输出固定值 |
| **Sine** | `waves` | 以正弦波形式在最小值和最大值之间波动 |
| **Ramp** | `trending_up` | 从最小值开始线性递增，到达最大值后重置 |

### 4.3 Modbus TCP 协议配置

Modbus TCP 协议用于通过 TCP/IP 网络与 Modbus 从站设备通信。

#### 配置参数

| 参数 | 必填 | 说明 | 默认值 | 验证规则 |
|------|------|------|--------|---------|
| **主机地址** | 是 | 从站设备的 IP 地址或域名 | 无 | 有效的 IP 或域名格式 |
| **端口** | 是 | Modbus TCP 服务端口 | 502 | 范围 1–65535 |
| **从站ID** | 是 | 从站设备地址 | 1 | 范围 1–247 |
| **超时 (ms)** | 否 | 通信超时时间（毫秒） | 5000 | 范围 100–60000 |
| **连接池大小** | 否 | 并发连接数 | 4 | 范围 1–32 |

#### 配置示例

```json
{
  "host": "192.168.1.100",
  "port": 502,
  "slave_id": 1,
  "timeout_ms": 5000,
  "connection_pool_size": 4
}
```

#### 连接测试

配置 Modbus TCP 参数后，可以点击对话框底部的 **⚡ 测试连接** 按钮验证配置是否正确：

| 测试状态 | 按钮显示 | 说明 |
|---------|---------|------|
| 未测试 | `⚡ 测试连接` | 初始状态 |
| 测试中 | `⟳ 测试中...` | 正在尝试连接设备 |
| 测试成功 | `✓ 连接成功 · 延迟 15ms` | 连接正常，显示响应延迟 |
| 测试失败 | `✗ 连接失败: 连接超时` | 连接异常，可重试 |

> 测试成功状态会在 5 秒后自动恢复为初始状态。

### 4.4 Modbus RTU 协议配置

Modbus RTU 协议用于通过串口 (RS485/RS232) 与 Modbus 从站设备通信。

#### 配置参数

| 参数 | 必填 | 说明 | 可选值 | 默认值 |
|------|------|------|--------|--------|
| **串口** | 是 | 系统串口设备路径 | 扫描结果或手动选择 | 无 |
| **波特率** | 是 | 串口通信速率 | 9600 / 19200 / 38400 / 57600 / 115200 | 9600 |
| **数据位** | 是 | 每个字符的数据位数 | 7 / 8 | 8 |
| **停止位** | 是 | 停止位长度 | 1 / 2 | 1 |
| **校验** | 是 | 奇偶校验方式 | None / Even / Odd | None |
| **从站ID** | 是 | 从站设备地址 | 1–247 | 1 |
| **超时 (ms)** | 否 | 通信超时时间 | 100–60000 | 1000 |

#### 串口扫描

配置 Modbus RTU 时，建议先使用 **📡 扫描串口** 功能自动检测系统可用串口：

| 扫描状态 | 按钮显示 | 说明 |
|---------|---------|------|
| 未扫描 | `📡 扫描串口` | 初始状态 |
| 扫描中 | `⟳ 扫描中...` | 正在检测串口设备 |
| 扫描完成 | `✓ 扫描完成` | 已发现串口，自动填入下拉列表 |
| 无设备 | `未检测到串口` | 未发现可用串口 |

#### 跨平台串口路径示例

| 平台 | 常见路径 |
|------|---------|
| **Linux** | `/dev/ttyUSB0`, `/dev/ttyACM0`, `/dev/ttyS0` |
| **macOS** | `/dev/cu.usbserial-*`, `/dev/tty.usbserial-*` |
| **Windows** | `COM1`, `COM2`, `COM3` |

#### 配置示例

```json
{
  "port": "/dev/ttyUSB0",
  "baud_rate": 9600,
  "data_bits": 8,
  "stop_bits": 1,
  "parity": "None",
  "slave_id": 1,
  "timeout_ms": 1000
}
```

### 4.5 编辑设备

1. 在工作台详情页，点击设备卡片或右侧的更多菜单 (⋮)，选择 **编辑**。
2. 弹出编辑对话框，基本信息和协议参数已预填当前值。
3. **协议类型** 在编辑模式下不可更改。
4. 修改后点击 **保存**。

### 4.6 删除设备

1. 在工作台详情页，点击设备卡片或右侧的更多菜单 (⋮)，选择 **删除**。
2. 确认后设备和所有关联测点将被删除。

## 5. 测点配置

测点 (Point) 是设备上的具体数据采集位置。每个设备可配置多个测点。

### 5.1 测点配置位置

在创建设备对话框的底部，有一个**测点配置**区域。也可以在设备详情页中为已有设备添加测点。

### 5.2 Virtual 测点配置

当设备协议为 Virtual 时，测点表格包含以下列：

| 列名 | 说明 | 示例 |
|------|------|------|
| **名称** | 测点标识名称 | "温度通道1" |
| **数据类型** | 测点数据类型 | Number / Integer / String / Boolean |
| **访问类型** | 读写权限 | RO / WO / RW |
| **最小值** | 数值范围下限 | 0.0 |
| **最大值** | 数值范围上限 | 100.0 |
| **固定值** | 仅 Fixed 模式有效 | 50.0 |
| **操作** | 删除该测点 | 🗑️ |

### 5.3 Modbus 测点配置

当设备协议为 Modbus TCP 或 Modbus RTU 时，测点表格包含以下列：

| 列名 | 说明 | 可选值/示例 |
|------|------|-----------|
| **名称** | 测点标识名称 | "线圈1状态" |
| **功能码** | Modbus 功能码 | 01 / 02 / 03 / 04 |
| **地址** | 寄存器/线圈起始地址 | 0–65535 |
| **数据类型** | 数据解析类型 | uint16, int16, float32 等 |
| **数量** | 连续读取的寄存器数量 | 1 |
| **缩放** | 原始值缩放因子 | 1.0 |
| **偏移** | 缩放后偏移量 | 0.0 |
| **操作** | 删除该测点 | 🗑️ |

#### Modbus 功能码说明

| 功能码 | 名称 | 数据类型 | 读写权限 | 说明 |
|--------|------|---------|---------|------|
| **01** | 线圈 (Coil) | 布尔值 | 读写 (RW) | 开关量输出 |
| **02** | 离散输入 (Discrete Input) | 布尔值 | 只读 (RO) | 开关量输入 |
| **03** | 保持寄存器 (Holding Register) | uint16 (16位) | 读写 (RW) | 通用数据存储 |
| **04** | 输入寄存器 (Input Register) | uint16 (16位) | 只读 (RO) | 模拟量输入 |

#### 数据转换公式

采集到的原始值会按以下公式转换：

```
显示值 = 原始值 × 缩放 + 偏移
```

**示例**: 原始寄存器值为 2500，缩放 0.01，偏移 0 → 显示值 = 2500 × 0.01 = 25.00

### 5.4 添加与删除测点

- **添加测点**: 点击测点表格下方的 **+ 添加测点** 按钮，新增一行。
- **删除测点**: 点击行尾的 🗑️ 删除图标。
- 添加/删除操作在表格内即时展示，使用平滑的展开/收缩动画。

## 6. 设备连接与状态

### 6.1 设备连接

在设备详情页的每个设备卡片上，都有一个连接状态按钮：

| 状态 | 按钮样式 | 指示灯 | 说明 |
|------|---------|--------|------|
| **已断开** | `连接` (轮廓按钮) | 🔴 红色圆点 | 设备未连接 |
| **连接中** | `连接中...` (禁用) | 🔵 旋转动画 | 正在建立连接 |
| **已连接** | `断开` (轮廓按钮) | 🟢 绿色圆点 | 设备已连接，数据正常 |
| **连接失败** | `重试` (轮廓按钮) | 🔴 红色圆点 | 连接失败，可重试 |

### 6.2 测点数据展示

设备连接后，测点表格实时展示数据：

```
┌─────────────────────────────────────────────────────────┐
│ [📡] 温度传感器1  Modbus TCP       ● 已连接  [断开连接]  │
│                                                         │
│ ┌────────┬────────┬──────────┬──────┬──────┬──────────┐ │
│ │ 名称   │ 功能码  │ 数据类型  │ 值   │ 缩放  │ 状态     │ │
│ ├────────┼────────┼──────────┼──────┼──────┼──────────┤ │
│ │ 温度1  │ 03 保持 │ uint16   │ 2500 │ 0.01 │ ● 正常   │ │
│ │ 温度2  │ 03 保持 │ uint16   │ 3000 │ 0.01 │ ● 正常   │ │
│ │ 压力1  │ 04 输入 │ uint16   │ 1500 │ 1.0  │ ● 正常   │ │
│ │ 状态1  │ 01 线圈 │ bool     │ true │ --   │ ● 正常   │ │
│ └────────┴────────┴──────────┴──────┴──────┴──────────┘ │
└─────────────────────────────────────────────────────────┘
```

#### 测点状态说明

| 状态图标 | 含义 | 说明 |
|---------|------|------|
| 🟢 绿色圆点 | 正常 | 数据读取正常 |
| 🟡 黄色圆点 | 警告 | 数据超过预设阈值 |
| 🔴 红色圆点 | 错误 | 数据读取失败 |
| ⚪ 灰色圆点 | 未连接 | 设备未连接，无数据 (`--`) |

#### 数据更新

- 设备连接后，测点值按配置的采样周期自动刷新。
- 值更新时有短暂的高亮闪烁效果 (500ms)，帮助您注意到数据变化。

### 6.3 断开连接

点击设备卡片上的 **断开** 按钮即可断开与设备的连接。断开后：
- 测点数据显示为 `--`
- 状态指示灯变为灰色
- 系统不再向该设备发送读写请求

## 7. 常见问题

### 7.1 登录相关

**Q: 忘记密码怎么办？**

目前请联系系统管理员重置密码。后续版本将支持自助密码重置功能。

**Q: 登录时提示"会话已过期"是什么意思？**

您的登录会话已超时 (默认 15 分钟无操作)，请刷新页面重新登录。

**Q: 登录后页面空白或加载失败？**

1. 检查后端服务是否正常运行 (`http://localhost:8080/api/v1/health`)。
2. 检查浏览器控制台是否有网络错误。
3. 尝试清除浏览器缓存后重新登录。

---

### 7.2 设备相关

**Q: Virtual 设备与其他协议设备的区别是什么？**

Virtual 设备是完全模拟的，不依赖任何物理硬件，数据由平台自动生成。适合开发测试、功能调试和用户培训。Modbus TCP/RTU 设备需要实际硬件或模拟器。

**Q: Modbus TCP 连接测试失败，如何排查？**

1. 确认目标设备的 IP 地址和端口是否正确。
2. 使用 `ping` 命令检查网络连通性。
3. 确认目标设备的防火墙是否允许 502 端口通信。
4. 确认从站 ID 与目标设备一致。
5. 检查网络超时设置是否合理（局域网建议 1000–5000ms）。

**Q: Modbus RTU 扫描不到串口怎么办？**

1. 确认串口设备已正确连接到计算机。
2. Linux 用户：检查 `/dev/ttyUSB*` 或 `/dev/ttyACM*` 是否存在，使用 `ls -la /dev/tty*` 查看。
3. macOS 用户：检查 `/dev/cu.usbserial-*` 是否存在。
4. Windows 用户：在设备管理器中确认串口设备已正确安装驱动。
5. 确认当前用户有串口读写权限（Linux 需加入 `dialout` 组: `sudo usermod -a -G dialout $USER`）。

**Q: 波特率、数据位、停止位、校验怎么设置？**

这些参数必须与连接的实际串口设备一致，请参考设备的产品手册。常见的 Modbus RTU 默认配置为: **9600, 8, 1, None**。

---

### 7.3 测点相关

**Q: 什么是功能码？为什么需要选择功能码？**

功能码是 Modbus 协议的核心概念，它指定了对设备执行的操作类型。例如：
- 功能码 03 (保持寄存器) 用于读写设备内部的 16 位数据。
- 功能码 01 (线圈) 用于控制开关量输出。

选择错误的功能码将导致无法正确读写数据。

**Q: 地址从哪里开始？0 还是 1？**

Kayak 使用 **0-based 寻址**，即地址从 0 开始。部分 Modbus 设备厂商使用 1-based 寻址，请查阅设备手册确认。如设备手册是 1-based，填入地址时请**减 1**。

**Q: 缩放和偏移怎么使用？**

用于将设备返回的原始整数值转换为实际物理量。例如：
- 温度传感器返回 0–65535 对应 0°C–100°C，则缩放 = `100/65535 ≈ 0.001526`。
- 或者使用 Modbus 设备内置工程值转换，将缩放设为 0.1（如果设备手册标注"值 = 原始值 × 0.1°C"）。

**Q: 为什么有些测点值显示为 `--`？**

这表示设备未连接或该测点读取失败。请先连接设备，然后检查该测点的功能码、地址和数据类型是否正确。

### 7.4 工作台相关

**Q: 一个工作台可以添加多少个设备？**

理论上没有硬性限制，但建议每个工作台的设备数不超过 50 个，以确保良好的界面响应性能。

**Q: 删除工作台后数据能恢复吗？**

不能。删除工作台是**不可逆**操作，所有关联的设备和数据将被永久删除。删除前请仔细确认。

**Q: 如何在不同工作台之间迁移设备？**

目前不支持直接迁移。您可以先在目标工作台创建相同配置的设备，然后删除原设备。后续版本将支持设备迁移功能。
