# Release 1 - Kayak v0.2.0

**版本**: 0.2.0  
**发布日期**: 2026-05-03  
**状态**: 正式发布

---

## 概述

Release 1（v0.2.0）在 Release 0 基础架构之上，新增了 Modbus 工业协议支持、连接池管理、Flutter Web 前端，以及开发测试用的 Modbus TCP 模拟设备 CLI 工具。

---

## 新功能

### 1. Modbus TCP 协议驱动

完整的 Modbus TCP 协议栈实现，支持以下操作：

- **线圈 (Coils)** — 读写 FC01/FC05/FC15
- **离散输入 (Discrete Inputs)** — 读取 FC02
- **保持寄存器 (Holding Registers)** — 读写 FC03/FC06/FC16
- **输入寄存器 (Input Registers)** — 读取 FC04

特性：
- MBAP 帧头封装/解封装
- PDU 构建和解析
- 自动重连机制
- 连接健康检查

### 2. Modbus RTU 协议驱动

通过 RS-232/RS-485 串行接口连接物理 Modbus RTU 设备：

- CRC16 校验
- 可配置的串口参数（波特率、数据位、停止位、校验）
- 与 TCP 驱动共享相同的功能码层

### 3. 连接池管理

高效的多路复用连接池：

- **Semaphore + VecDeque** 实现，支持并发连接复用
- 连接数量可配置的上限管理
- 自动空闲连接回收
- 连接健康检查与自动恢复
- 防泄漏机制（guard-based 归还）

### 4. Modbus TCP 模拟设备 CLI

`modbus-simulator` 是一个命令行工具，用于在开发测试中模拟 Modbus TCP 从设备：

```bash
# 默认端口 5020
cargo run --bin modbus-simulator

# 自定义端口
cargo run --bin modbus-simulator -- --port 5021
```

模拟设备提供可配置的虚拟寄存器组，支持动态数据更新。

### 5. Flutter Web 前端

基于 Flutter 构建的 Web 应用：

- **Material Design 3** 设计系统，主色 `#1976D2`
- 响应式布局，适配桌面和移动浏览器
- 工作台-设备-测点三层管理界面
- 设备配置与连接状态监控
- 实时数据仪表盘
- 认证状态管理与路由守卫
- **访问地址**: `http://localhost:8080`

### 6. 设备连接测试 API

新增设备连接诊断功能：

- `POST /api/v1/devices/{id}/connect` — 连接设备
- `POST /api/v1/devices/{id}/disconnect` — 断开设备
- `POST /api/v1/devices/{id}/test` — 测试设备连接

### 7. 配置点管理增强

- 测点配置持久化存储
- 测点初始值设置
- 测点读写属性检查

## 升级说明

### 从 v0.1.0 升级到 v0.2.0

```bash
# 1. 拉取最新代码
git checkout main
git pull origin main

# 2. 重新构建
cd kayak-backend && cargo build --release
cd ../kayak-frontend && flutter build web --release

# 3. 启动服务
cd ..
./scripts/start-web.sh
```

### 数据库兼容性

v0.2.0 对数据库 schema 有增量变更（新增设备连接参数列），**从 v0.1.0 升级无需迁移**，新字段会自动填充默认值。

如果需要全新安装：

```bash
rm -rf data/kayak.db
./scripts/start-web.sh
```

### 依赖变更

- **tokio-modbus** — Modbus TCP/RTU 协议底层库
- **tokio-serial** — 串口支持（Modbus RTU）
- 前端无新依赖

## 已知问题

| 编号 | 问题 | 影响 | 计划修复 |
|------|------|------|----------|
| R1-01 | Modbus RTU 仅在 Linux/macOS 串口环境测试 | Windows 串口兼容性待验证 | v0.3.0 |
| R1-02 | 连接池在高并发场景下未做压力测试 | >100 并发连接可能需要调优 | v0.3.0 |
| R1-03 | Web 前端首次加载时间较长（~5MB） | 首次访问等待 3-5 秒 | v0.3.0 (代码分割) |
| R1-04 | 模拟设备不支持 Modbus RTU 模式 | 仅提供 TCP 模拟 | v0.3.0 |

## API 端点变更

### 新增端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/devices/{id}/connect` | POST | 连接设备 |
| `/api/v1/devices/{id}/disconnect` | POST | 断开设备连接 |
| `/api/v1/devices/{id}/test` | POST | 测试设备连接 |

### 修改端点

| 端点 | 变更 |
|------|------|
| `POST /api/v1/devices` | 新增 `protocol_type: "modbus-tcp"` / `"modbus-rtu"` 选项 |
| `GET /api/v1/devices/{id}/points` | 新增测点实时值读取字段 |
