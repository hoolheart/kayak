
## 8. 经验教训

1. **子智能体步骤限制**: 复杂任务（如代码审查、详细设计）容易达到步骤上限，需要拆分为更小的子任务
2. **环境问题**: `libudev-dev` 缺失导致后端编译验证无法进行，建议在 CI 环境中补充验证
3. **并行开发效率**: 前后端并行开发（后端 API + 前端图表）显著提升了 Sprint 效率
4. **Mock 数据策略**: 前端使用 Mock 数据独立开发，避免了等待后端 API 的阻塞
