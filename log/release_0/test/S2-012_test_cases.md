## 1. 后端API测试用例

### 1.1 方法列表查询 API

#### TC-S2-012-BE-001: 获取方法列表 - 成功
- **测试描述**: 获取用户的方法列表
- **请求**: `GET /api/v1/methods?page=1&size=10`
- **前置条件**: 用户已登录，数据库中有2个方法
- **预期结果**:
  - 状态码 200
  - 返回 `{code: 200, data: {items: [...], total: 2, page: 1, size: 10}}`
  - items包含2个方法对象
  - 每个方法对象包含id, name, description, process_definition, parameter_schema, version, created_by, created_at, updated_at

#### TC-S2-012-BE-002: 获取方法列表 - 空列表

- **预期结果**:
  - 状态码 200
  - 返回 `{code: 200, data: {items: [], total: 0, page: 1, size: 10}}`

#### TC-S2-012-BE-003: 获取方法列表 - 分页
- **测试描述**: 分页查询方法列表

  - 状态码 200

#### TC-S2-012-BE-004: 获取方法列表 - 未认证

### 1.3 创建方法 API

#### TC-S2-012-BE-007: 创建方法 - 成功
- **测试描述**: 创建一个新的试验方法
- **请求**: `POST /api/v1/methods`
- **请求体**:
  ```json
  {
    "name": "温度测试方法",
    "description": "测试温度变化过程",
    "process_definition": {
      "nodes": [
        {"id": "start", "type": "Start"},
        {"id": "read_temp", "type": "Read", "point_id": "point_1", "variable": "temp"},
        {"id": "end", "type": "End"}
      ]
    },
    "parameter_schema": {
      "temperature_setpoint": {
        "type": "number",
        "default": 25.0,
        "unit": "°C"
      }
    }
  }
  ```
- **预期结果**:
  - 状态码 201
  - 返回创建的方法对象，包含生成的id和version=1

#### TC-S2-012-BE-008: 创建方法 - 名称为空
- **测试描述**: 创建方法时名称为空
- **请求**: `POST /api/v1/methods`
- **请求体**: `{"name": "", "process_definition": {}, "parameter_schema": {}}`
- **预期结果**: 状态码 400，错误信息包含"名称"

#### TC-S2-012-BE-009: 创建方法 - 名称过长
- **测试描述**: 创建方法时名称超过255字符
- **请求**: `POST /api/v1/methods`
- **请求体**: `{"name": "a".repeat(256), "process_definition": {}, "parameter_schema": {}}`
- **预期结果**: 状态码 400

#### TC-S2-012-BE-010: 创建方法 - 过程定义非对象
- **测试描述**: 过程定义不是JSON对象
- **请求**: `POST /api/v1/methods`
- **请求体**: `{"name": "test", "process_definition": [], "parameter_schema": {}}`
- **预期结果**: 状态码 400，错误信息包含"过程定义"

#### TC-S2-012-BE-011: 创建方法 - 参数Schema非对象
- **测试描述**: 参数Schema不是JSON对象
- **请求**: `POST /api/v1/methods`

### 1.6 方法验证 API

#### TC-S2-012-BE-018: 验证方法 - 有效过程定义
- **测试描述**: 验证一个有效的过程定义
- **请求**: `POST /api/v1/methods/validate`
- **请求体**:
  ```json
  {
    "process_definition": {
      "nodes": [
        {"id": "start", "type": "Start"},
        {"id": "end", "type": "End"}
      ]
    }
  }
  ```
- **预期结果**: 状态码 200，返回 `{valid: true, errors: []}`

#### TC-S2-012-BE-019: 验证方法 - 缺少Start节点
- **测试描述**: 验证缺少Start节点的过程定义
- **请求**: `POST /api/v1/methods/validate`
- **请求体**:
  ```json
  {
    "process_definition": {
      "nodes": [
        {"id": "end", "type": "End"}
      ]
    }
  }
  ```
- **预期结果**: 状态码 200，返回 `{valid: false, errors: ["缺少Start节点"]}`

#### TC-S2-012-BE-020: 验证方法 - 缺少End节点
- **测试描述**: 验证缺少End节点的过程定义
- **请求**: `POST /api/v1/methods/validate`
- **请求体**:
  ```json
  {
    "process_definition": {
      "nodes": [
        {"id": "start", "type": "Start"}
      ]
    }
  }
  ```
- **预期结果**: 状态码 200，返回 `{valid: false, errors: ["缺少End节点"]}`

#### TC-S2-012-BE-021: 验证方法 - 无效节点类型
- **测试描述**: 验证包含无效节点类型的过程定义
- **请求**: `POST /api/v1/methods/validate`
- **请求体**:
  ```json
  {
    "process_definition": {
      "nodes": [
        {"id": "start", "type": "Start"},
        {"id": "invalid", "type": "InvalidType"},
        {"id": "end", "type": "End"}
      ]
    }
  }
  ```
- **预期结果**: 状态码 200，返回 `{valid: false, errors: ["节点'invalid'的类型'InvalidType'无效"]}`
