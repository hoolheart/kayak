## 关键问题 (Critical Issues)

### 1. 组件类型不一致 (TC-S1-011-03 vs TC-S1-011-04)

**位置**: TC-S1-011-03 和 TC-S1-011-04

**问题描述**:
- TC-S1-011-03 期望 `ElevatedButton`
- TC-S1-011-04 期望 `FilledButton` (MD3 组件)

这两个测试用例对按钮组件的期望不一致。Material Design 3 规范主要使用 `FilledButton` 或 `FilledButton.tonal`，`ElevatedButton` 是 MD2 组件。

**建议**:
```dart
// TC-S1-011-03 应修改为:
expect(find.widgetWithText(FilledButton, RegExp(r'Login|登录')), findsOneWidget);
```

**优先级**: P0

---

### 2. 缺少登录失败后停留在登录页的测试

**位置**: 第 6 节 (导航测试)

**问题描述**: 
- 验收标准 AC3: "登录成功跳转首页"
- 当前测试用例覆盖了登录成功跳转 (TC-S1-011-14, TC-S1-011-23)
- **缺失**: 登录失败后用户是否仍停留在登录页

**建议添加测试用例**:
```markdown
### TC-S1-011-36: 登录失败停留在登录页测试
| 字段 | 内容 |
|------|------|
| **用例ID** | TC-S1-011-36 |
| **用例名称** | 登录失败停留在登录页测试 |
| **测试类型** | Integration Test |
| **优先级** | P0 |
| **前置条件** | 登录页面加载完成 |
| **测试步骤** | 1. 输入无效凭证 2. 点击登录 3. 等待响应 |
| **预期结果** | 路由保持 /login，不跳转 |
| **自动化代码** | `expect(router.currentLocation, equals('/login'));` |
```

**优先级**: P0

---

### 3. TC-S1-011-09 前置条件不足

**位置**: TC-S1-011-09 (验证错误时按钮禁用测试)

**问题描述**: 测试步骤只说"存在验证错误"，但没有明确**如何触发**验证错误。

**建议修改测试步骤**:
```markdown
| **测试步骤** | 1. 输入无效邮箱 "test@"<br>2. 触发验证 (离开输入框或提交表单)<br>3. 检查登录按钮状态 |
```

**优先级**: P1

---

### 4. TC-S1-011-21 缺少实现细节

**位置**: TC-S1-011-21 (主题切换测试)

**问题描述**: 自动化代码中引用 `themeNotifier`，但测试步骤中没有说明如何获取 `themeNotifier`。

**当前代码**:
```dart
await themeNotifier.toggleTheme();
expect(newBrightness, equals(Brightness.dark));
```

**建议**: 添加如何访问 themeNotifier 的说明，或改用以下方式:
```dart
// 通过 Provider 获取
final themeNotifier = container.read(themeProvider.notifier);
await themeNotifier.toggleTheme();
// 然后重建 widget 验证
```

**优先级**: P1

**位置**: TC-S1-011-13, TC-S1-011-17

**建议**: 添加 mock 说明:
