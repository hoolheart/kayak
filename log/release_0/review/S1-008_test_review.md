#### ✅ 一致项

| username字段 | 注册请求中包含 username，但架构 users 表有此字段，确认是否必需 |

## 5. 测试执行建议

### 5.1 测试环境配置

根据测试用例要求，建议按以下顺序搭建环境：

1. **单元测试**: 无需外部依赖，使用 `cargo test` 直接运行
2. **集成测试**: 需要内存 SQLite 或测试 PostgreSQL 实例
3. **API测试**: 需要启动完整后端服务
4. **安全测试**: 建议使用专门的安全测试工具（如 OWASP ZAP）辅助

### 5.2 测试执行顺序

```
Phase 1: 单元测试（无需服务）
  ├── TC-S1-008-10: Access Token结构
  ├── TC-S1-008-11: Refresh Token结构  
  ├── TC-S1-008-12: Token过期验证
  ├── TC-S1-008-13: Token签名验证
  └── TC-S1-008-16: bcrypt哈希

Phase 2: 集成测试（需要测试数据库）
  ├── TC-S1-008-01: 注册成功
  ├── TC-S1-008-02: 邮箱已存在
  └── TC-S1-008-06: 登录成功

Phase 3: API测试（需要完整服务）
  ├── TC-S1-008-03: 无效邮箱格式

  ├── TC-S1-008-07: 邮箱不存在
```
