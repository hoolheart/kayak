## 1. 测试概述

### 1.1 测试目标

验证 Modbus 设备创建设备后可配置测点的完整功能，包括：

1. **功能码选择**：FC01(Coil)、FC02(Discrete Input)、FC03(Holding Register)、FC04(Input Register) 的下拉选择及功能码与 Modbus 数据模型映射
2. **地址范围验证**：0-65535 的数值输入验证，非法值拦截
3. **数量验证**：1-125 的数值输入验证，以及数量与数据类型的联动（float32 占用 2 个寄存器）
4. **数据类型选择**：uint16/int16/float32 与功能码的兼容性约束（FC01/FC02 仅支持布尔型，FC03/FC04 支持 uint16/int16/float32）
5. **缩放因子/偏移量**：数值输入验证、默认值 1.0/0.0、正/负/零值支持
6. **测点列表增删**：添加测点到列表、编辑列表中测点、从列表删除测点、重复测点检测
7. **表单提交**：完整配置提交后的数据格式验证

### 1.2 Modbus 功能码与数据模型对应关系

| 功能码 | 功能码值 | 数据区域 | 数据类型 | 访问权限 |
|--------|---------|---------|---------|---------|
| FC01 | 0x01 | Coil (线圈) | 布尔 (bool) | RW (读写) |
| FC02 | 0x02 | Discrete Input (离散输入) | 布尔 (bool) | RO (只读) |
| FC03 | 0x03 | Holding Register (保持寄存器) | uint16/int16/float32 | RW (读写) |
| FC04 | 0x04 | Input Register (输入寄存器) | uint16/int16/float32 | RO (只读) |

### 1.3 PRD 测点配置 JSON 结构

```json
{
  "modbus": {
    "function_code": 3,
    "address": 0,
    "quantity": 1,
    "data_type": "uint16",
    "scale": 1.0,
    "offset": 0.0
  }
}
```

### 1.4 约束规则摘要

| 规则 | 约束 |
|------|------|
| FC01/FC02 数据类型 | 仅可配置为布尔型 (BOOL) |
| FC01/FC02 数量 | quantity 仍是 1-125（线圈/离散输入一次最多读 2000，但 UI 限制为 125） |
| FC03/FC04 数据类型 | 可选 uint16, int16, float32 |
| 地址范围 | 0 - 65535 |
| 数量范围 | 1 - 125 |
| 数量与 float32 | float32 时，数量 × 2 ≤ 125 (每个 float32 占 2 个寄存器) |
| float32 地址+寄存器联合边界 | address + quantity × 2 ≤ 65536 |
| 地址 + 数量边界 | address + quantity ≤ 65536 |
| 缩放因子 | 浮点数，可正可负可零（默认 1.0） |
| 偏移量 | 浮点数，可正可负可零（默认 0.0） |

| **覆盖代码** | `modbusAddressQuantity` 联合验证逻辑 `address + quantity ≤ 65536`，确认使用 `≤` 而非 `<` |

### 9.4 依赖的验证器扩展

以下验证器需在 `device_validators.dart` 中新增：

| 验证器方法 | 用途 | 约束 |
|-----------|------|------|
| `modbusAddress(String?)` | 验证 Modbus 地址 | 0-65535 整数 |
| `modbusQuantity(String?)` | 验证测点数量 | 1-125 整数 |
| `modbusQuantityForFloat32(String?, String?)` | float32 数量双重验证 | quantity × 2 ≤ 125 且 address + quantity × 2 ≤ 65536 |
| `modbusAddressQuantity(String?, String?)` | 联合验证地址+数量（非 float32） | address + quantity ≤ 65536 |
| `modbusScale(String?)` | 缩放因子验证 | 有效浮点数 |
| `modbusOffset(String?)` | 偏移量验证 | 有效浮点数 |

### 9.5 依赖的数据模型扩展

`point.dart` 的 `Point` 模型 `metadata` 字段需支持嵌套的 `ModbusPointConfig` 结构：

```dart
class ModbusPointConfig {
  final int functionCode;   // 1, 2, 3, 4
  final int address;        // 0-65535
  final int quantity;        // 1-125 (or 1-62 for float32)
  final String dataType;    // "uint16", "int16", "float32"
  final double scale;       // default 1.0
  final double offset;      // default 0.0
}
```
