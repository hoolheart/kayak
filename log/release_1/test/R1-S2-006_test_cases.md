## 1. 测试概述

### 1.1 测试目标

验证 Modbus TCP 模拟设备的 CLI 独立运行工具 `modbus-simulator`，确保：

1. **CLI 启动正确性**：命令行参数解析正确，能按预期绑定端口、设定从站 ID、加载初始值文件
2. **Modbus TCP 协议正确性**：作为 Modbus TCP 从站，正确响应 FC01 (Read Coils) 和 FC03 (Read Holding Registers) 请求
3. **错误处理健壮性**：非法参数、端口占用、无效配置文件等异常情况有清晰错误提示，不会崩溃
4. **启动信息输出**：启动时输出监听端口和寄存器状态信息，便于用户确认

### 1.2 被测工具

**命令**：

```
cargo run --bin modbus-simulator [OPTIONS]
```

或直接执行编译产物：

```
./target/debug/modbus-simulator [OPTIONS]
```

**预期 CLI 参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--port` | u16 | `502` | 监听端口号 |
| `--slave-id` | u8 | `1` | Modbus 从站 ID (1-247) |
| `--init-file` | Path | 无 (使用默认值) | 初始值 JSON 配置文件路径 |
| `--help` / `-h` | flag | — | 打印帮助信息 |
| `--version` / `-V` | flag | — | 打印版本信息 |

### 1.3 模拟设备默认内存布局

| 内存区域 | 默认大小 | 默认值 | 可读写 |
|---------|---------|--------|--------|
| Coils (线圈) | 100 (地址 0-99) | 全部 `false` | RW |
| Discrete Inputs (离散输入) | 100 (地址 0-99) | 全部 `false` | RO |
| Holding Registers (保持寄存器) | 100 (地址 0-99) | 全部 `0` | RW |
| Input Registers (输入寄存器) | 100 (地址 0-99) | 全部 `0` | RO |

### 1.4 测试范围限制

根据 R1-S2-006-A 的验收标准，**本次测试仅覆盖 FC01 (Read Coils) 和 FC03 (Read Holding Registers)**。其他功能码 (FC02/FC04/FC05/FC06/FC15/FC16) 已在 R1-S1-003 测试中覆盖核心库，不在此 CLI 工具测试范围内重复。

| 属性 | 内容 |
|------|------|
| **测试ID** | CS-001 |
| **优先级** | P0 (Critical) |
| **类别** | CLI 启动 / 冒烟测试 |
| **描述** | 不带任何参数启动，验证默认绑定到端口 502 并正常监听 |

## 7. 参考数据

### 7.1 默认初始值文件 (完整模板)

```json
{
  "coils": {
    "0": false,
    "1": false
  },
  "discrete_inputs": {
    "0": false
  },
  "holding_registers": {
    "0": 0,
    "1": 0
  },
  "input_registers": {
    "0": 0
  }
}
```

### 7.2 Modbus FC01 请求帧示例

**请求 (读线圈 0, 数量=1)**：
```
MBAP:  00 01 00 00 00 06 01
PDU:   01 00 00 00 01
```
- 事务ID: 0x0001
- 协议ID: 0x0000
- 长度: 0x0006
- 单元ID: 0x01
- 功能码: 0x01
- 起始地址: 0x0000
- 数量: 0x0001

**正常响应 (coil 0=false)**：
```
MBAP:  00 01 00 00 00 04 01
PDU:   01 01 00
```
- 字节数: 0x01
- 数据: 0x00 (coil 0 = false)

### 7.3 Modbus FC03 请求帧示例

**请求 (读保持寄存器 0, 数量=1)**：
```
MBAP:  00 01 00 00 00 06 01
PDU:   03 00 00 00 01
```
- 功能码: 0x03
- 起始地址: 0x0000
- 数量: 0x0001

**正常响应 (register 0 = 1234)**：
```
MBAP:  00 01 00 00 00 05 01
PDU:   03 02 04 D2
```
- 字节数: 0x02
- 数据: 0x04D2 (1234 decimal)

### 7.4 Modbus 异常响应格式

**功能码 = FC01, 异常 = Illegal Data Address**：
```
MBAP:  00 01 00 00 00 03 01
PDU:   81 02
```
- 功能码: 0x81 (FC01 + 0x80)
- 异常码: 0x02 (Illegal Data Address)

## 8. 风险评估

| 风险 | 影响 | 可能性 | 缓解措施 |
|------|------|--------|---------|
| 端口 502 被系统服务占用（如系统 Modbus） | 高 | 中 | CS-001 使用默认端口前先检查；提供 `--port` 回退方案 |
| OS 端口释放延迟 (TIME_WAIT) 影响 BR-003 | 中 | 高 | 快速启停测试间隔加 `sleep 1`；使用 SO_REUSEADDR |
| pymodbus 版本兼容性问题 | 中 | 低 | 固定 pymodbus 版本；提供备选 `mbpoll` 命令行工具 |
| Docker 环境端口映射问题 | 低 | 中 | Dockerfile 中 EXPOSE 502，测试使用 `-p` 明确映射 |
| 模拟设备不响应非本从站 ID 的请求 | 低 | 中 | 已在 CS-005 覆盖，确认行为一致性 |
| 初始值文件格式与核心库不兼容 | 高 | 中 | CS-006 和 EH-008 显式验证；文件格式应遵循 PRD 2.3.3 定义 |
