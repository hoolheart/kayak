## 5. `.gitignore` Analysis — TWO BUGS FOUND

### Bug A: Wrong directory path

The `.gitignore` has:
```
/test/golden_files/failures/
```

But the actual failure directory is:
```
/test/widget/golden/failures/
```

These are different paths. The `golden_files/` directory contains the committed baseline images — `failures/` would never appear there. The actual failure output goes to `test/widget/golden/failures/` (relative to the test file location).

### Bug B: Wrong filename patterns

The `.gitignore` has:
```
*.diff.png
*.masked.png
```

But Flutter's current golden test failure naming is:
- `*_isolatedDiff.png` (NOT `*.diff.png`)
- `*_maskedDiff.png` (NOT `*.masked.png`)
- `*_masterImage.png` (not excluded at all)
- `*_testImage.png` (not excluded at all)

**Verification**: `ls test/widget/golden/failures/*.diff.png` returns no matches. `ls test/widget/golden/failures/*.masked.png` returns no matches. The patterns are completely ineffective.

### Combined effect

Due to both bugs, **all 24 golden failure files are untracked** and will show up in `git status` for anyone running golden tests until the `.gitignore` is fixed.


## 6. Root Cause Analysis

The baselines were created by commit `cf55f6f` ("chore(test): add golden reference files"). They were likely generated on a **different platform** (possibly Linux/CI). The current environment (macOS) renders slightly differently, causing all 6 tests to fail with consistent but small pixel differences (0.15%–1.00%).

This is a well-known Flutter golden test challenge:
- Different OS → different font rendering
- Different Flutter version → different widget painting
- Different screen/DPI → different pixel output
